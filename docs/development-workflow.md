# 開発ワークフロー

## 採用した方針

miraclepopcornでは、実装仕様をリポジトリ内のMarkdownで管理し、ChatGPTとCodexを組み合わせて開発する。

GitHub Issueと`gh`コマンドを中心とした運用も検討したが、現時点では採用しない。主な理由は次のとおり。

- Codexに与えるGitHub権限をできるだけ小さくしたい。
- `gh`の認証や権限管理を開発フローに組み込みたくない。
- 実装仕様はIssueよりも、コードと一緒に管理できるリポジトリ内のMarkdownの方が扱いやすい。
- AIが参照する仕様を、実装と同じ変更単位で管理したい。
- GitHubは進捗管理、レビュー、履歴管理を主な役割とする。

これは現在のプロジェクト規模と権限方針に基づく設計判断であり、将来のチーム構成や運用要件に応じて見直す可能性がある。

## 現在の流れ

1. ChatGPTで仕様や実装内容を検討する。
2. 実装タスクを `docs/tasks/` 配下のMarkdownにまとめる。
3. タスクMarkdownをリポジトリに置く。
4. CodexがタスクMarkdownと既存コードを確認し、ローカルで実装する。
5. ChatGPTまたはCodexで差分やPull Requestをレビューする。

タスクの記述には [タスクテンプレート](tasks/template.md) を使い、UIやデザインに関わる変更は [デザイン方針](design.md) に従う。
